## ⚠️ Règles Essentielles

### 1. Commandes Symfony - TOUJOURS dans Docker
```bash
# ✅ CORRECT - Exécuter dans le conteneur Docker
docker-compose exec symfony php bin/console make:migration
docker-compose exec symfony php bin/console doctrine:migrations:migrate

# ❌ INCORRECT - Ne JAMAIS exécuter sur la machine hôte
php bin/console ...  # PHP n'est pas installé
composer ...
```

**Pourquoi ?** PHP n'est pas installé sur la machine hôte. ALL Symfony commands must run inside the Docker container.

### 2. Architecture du Projet

- **Localisation :** `/symfony/`
- **Port :** 8080
- **Base de données :** MySQL
- **API :** JSON-LD (API Platform)
- **Base URL API :** `http://localhost:8080/api`

#### Frontend (Angular)
- **Localisation :** `/angular/`
- **Port :** 4200 (dev), 80 (Docker)
- **Framework :** Angular 17+

#### Base de Données (MySQL)
- **Localisation :** `/docker/mysql_data/`
- **Base :** `greeting_card`

### 3. Modèles de Données Clés

#### Card Entity (Symfony)
```php
- id: int (primary key)
- title: string
- description: text
- picture: Picture (OneToOne)
- tags: Tag[] (ManyToMany)
- isActif: boolean (default: true)  // ← NOUVEAU
```

#### Card Model (Angular)
```typescript
interface Card {
  id?: number;
  title: string;
  description: string;
  picture: Picture;
  tags: Tag[];
}
```

### 4. API Communication Pattern

**Service:** `ConnectionApiService` (singleton)
- **Headers:** `Accept: application/ld+json`
- **Format réponse:** JSON-LD avec `hydra:member` pour les collections
- **Gestion d'état:** BehaviorSubjects + RxJS Observables
- **Unsubscription:** Pattern `takeUntil`

**Exemple de réponse API:**
```json
{
  "@context": "...",
  "@id": "/api/cards",
  "@type": "hydra:Collection",
  "hydra:member": [
    { "id": 1, "title": "...", ... }
  ]
}
```

### 7. Workflow de Migration

Quand tu ajoutes un champ à une Entity Symfony:

1. Modifie l'Entity (ex: `Card.php`)
2. Génère la migration (dans Docker):
   ```bash
   docker-compose exec symfony php bin/console make:migration
   ```
3. Exécute la migration:
   ```bash
   docker-compose exec symfony php bin/console doctrine:migrations:migrate
   ```
4. Mets à jour le modèle Angular si nécessaire
5. Ajoute les groupes de sérialisation `#[Groups(['card:read', 'card:write'])]`

```bash
  -d '{"title":"...", "description":"...", ...}'
```
